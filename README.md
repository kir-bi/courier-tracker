# Courier Tracker

B2B SaaS для ресторанов с собственной курьерской службой.

## Идея

Рестораны со своими курьерами не видят, где те находятся в моменте, и не сохраняют историю маршрутов. Клиентам отвечают «примерно через 40 минут» — без реальных данных.

**Courier Tracker** — простой инструмент трекинга:
1. Курьер открывает мобильное приложение → начинает смену → шарит геолокацию в фоне.
2. Менеджер в веб-админке видит на карте всех курьеров в реальном времени, может смотреть маршруты за день с детализацией.
3. Курьер пользуется внешней навигацией (Google Maps), а в нашем приложении только нажимает «Я приехал» — система автоматически выдаёт следующий заказ.

## Кому это нужно

Рестораны и фудкорты с собственной доставкой — 2-20 курьеров. **Не для агрегаторов** (Яндекс.Еда, Glovo) с собственной логистикой.

Первый клиент для валидации: **Виталий** (Kilogramm, Кайфуй — Полтава, Луцк, Сумы, Катовице).

## Поверхности продукта (MVP)

| Поверхность | Платформа | Для кого |
|-------------|-----------|----------|
| **Лендинг**       | веб      | потенциальный клиент |
| **Веб-админка**   | только веб (десктоп / планшет) | менеджер ресторана |
| **Приложение курьера** | iOS + Android | курьер |

**Чего нет в MVP:**
- ❌ Мобильное приложение для менеджера — только веб.
- ❌ Карта и навигация в приложении курьера — он пользуется Google Maps.
- ❌ Клиентский live-трекинг и SMS — это V2.

## Стек (гипотеза)

- Мобильное приложение: React Native (или нативно — решить после прототипа background location).
- Веб-админка: Next.js / React + Tailwind.
- Бэкенд: Node.js + PostgreSQL + Redis для real-time каналов (WebSocket).
- Карты: Mapbox или Google Maps.
